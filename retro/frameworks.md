# Retrospective Frameworks

3 battle-tested retro formats for cross-functional Agile teams.

## Overview

| Item | Detail |
|---|---|
| Frequency | End of every sprint |
| Duration | 1 hour max |
| Attendees | Full team |
| Output | 3 action items maximum |

---

## Rule #1 — Retros only work if they are psychologically safe

No blame. No hierarchy. No "who did this."
The PM sets the tone — if the team is afraid to speak, the retro is useless.

---

## Framework 1 — Start / Stop / Continue

Best for: teams new to retros, quick sessions

| Column | Question |
|---|---|
| Start | What should we start doing? |
| Stop | What should we stop doing? |
| Continue | What is working well? |

**How to run it:**
1. Give everyone 5 minutes to write sticky notes silently
2. Each person shares one item per column
3. Group similar items together
4. Vote on top 3 action items
5. Assign an owner to each action item

---

## Framework 2 — 4Ls

Best for: teams that need deeper reflection

| Column | Question |
|---|---|
| Liked | What did you enjoy this sprint? |
| Learned | What did you learn? |
| Lacked | What was missing? |
| Longed for | What do you wish you had? |

**How to run it:**
1. Silent writing — 7 minutes
2. Round robin sharing
3. Dot vote on top issues
4. Define action items with owners and due dates

---

## Framework 3 — Mad / Sad / Glad

Best for: teams going through difficult sprints

| Column | Question |
|---|---|
| Mad | What frustrated you? |
| Sad | What disappointed you? |
| Glad | What made you happy? |

**How to run it:**
1. Silent writing — 5 minutes
2. PM reads items anonymously if team is shy
3. Discuss patterns not individual incidents
4. Agree on 3 improvements for next sprint

---

## Action item template

| Action | Owner | Due |
|---|---|---|
| Define acceptance criteria before grooming | PM | Next sprint start |
| Add automated tests to deployment checklist | Engineering lead | Sprint 2 |
| Schedule design review earlier in sprint | Designer | Ongoing |

---

## Common mistakes to avoid

- Retros without action items — just venting
- Same format every sprint — teams go on autopilot
- No follow up on previous action items
- PM talking more than the team
