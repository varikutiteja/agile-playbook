# Agile Playbook

A practical Agile & Scrum process guide built from 4+ years of hands-on 
experience managing cross-functional product and engineering teams in 
Tech and FinTech environments.

## What's inside

| File | Description |
|---|---|
| `sprint-planning/guide.md` | How to run sprint planning end to end |
| `backlog/grooming.md` | Backlog grooming process and templates |
| `retro/frameworks.md` | 3 retrospective frameworks with templates |
| `user-stories/template.md` | User story format with real examples |
| `definition-of-done.md` | What "done" actually means |

## How I used this at VIGIT TECH

- Managed 3–5 concurrent Agile workstreams simultaneously
- Facilitated sprint planning, backlog grooming, standups and retros
- Reduced reconciliation errors by 30% through structured delivery
- Served as liaison between business stakeholders and engineering teams

## Core Agile principles I follow

- Ship small, learn fast
- Block removal is a PM's most important job
- Retros only work if they're psychologically safe
- A backlog without priorities is just a wish list
